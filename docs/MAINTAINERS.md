# Maintainers

Updated: 2026-06-05

This document defines maintainer responsibilities for `.ontos Protocol`.

## Maintainer Roles

Required roles:

- protocol maintainer
- parser maintainer
- CLI maintainer
- viewer maintainer
- documentation maintainer
- release maintainer
- security maintainer
- triage maintainer

## Responsibilities

Protocol maintainer:

- owns specification clarity
- reviews syntax proposals
- reviews AST changes
- reviews compatibility impact
- maintains field registry

Parser maintainer:

- owns parser correctness
- owns serializer and formatter behavior
- owns conformance fixtures
- owns diagnostics consistency

CLI maintainer:

- owns `ontosfmt`
- owns command behavior
- owns exit codes
- owns install and packaging quality

Viewer maintainer:

- owns browser viewer
- owns accessibility quality
- owns local-first behavior
- owns demo readiness

Documentation maintainer:

- owns README
- owns docs site
- owns examples gallery
- owns migration docs
- owns launch copy consistency

Release maintainer:

- owns release process
- owns package publishing
- owns GitHub releases
- owns release notes

Security maintainer:

- owns vulnerability intake
- owns dependency review
- owns public-boundary scan
- owns security release coordination

Triage maintainer:

- owns issue labels
- owns duplicate handling
- owns launch-week responses
- owns contributor routing

## Maintainer Expectations

Maintainers should:

- keep decisions documented
- prefer compatibility
- avoid private context in public decisions
- ask for RFCs when compatibility is affected
- keep public communication specific and respectful
- protect the local-first nature of the protocol
