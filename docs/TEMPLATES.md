# Templates

The first release ships official examples that can be copied as starting
templates for common project workflows.

## App Design

Use [`examples/app-design.ontos`](../examples/app-design.ontos) for product and
interface planning.

## Project State

Use [`examples/project-state.ontos`](../examples/project-state.ontos) for
current implementation state, risks, and verification.

## AI Handoff

Use [`examples/ai-handoff.ontos`](../examples/ai-handoff.ontos) when handing a
bounded task to an AI coding tool.

## Bug Fix

Use [`examples/bug-fix.ontos`](../examples/bug-fix.ontos) for defects,
reproduction steps, risks, and acceptance checks.

## Product Spec

Use [`examples/product-spec.ontos`](../examples/product-spec.ontos) for product
requirements and implementation boundaries.

## Release Checklist

Use [`examples/release-plan.ontos`](../examples/release-plan.ontos) for release
readiness, gates, and launch verification.

## Review Pack

Use [`examples/review-pack.ontos`](../examples/review-pack.ontos) for review
scope, risks, and evidence.

## Team Knowledge

Use [`examples/team-knowledge.ontos`](../examples/team-knowledge.ontos) for
shared decisions and operational memory.

## Research Notes

Use [`examples/research-notes.ontos`](../examples/research-notes.ontos) for
structured research and source notes.
